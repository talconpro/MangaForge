# MangaForge MVP 开发计划与任务拆解

版本：V0.5 MVP  
项目名称：MangaForge  
项目定位：AI 长篇小说漫画改编工作台  
前端技术栈：Vue 3 + Vite + TypeScript  
后端技术栈：Python FastAPI  
AI 执行层：HermesAgent / OpenAI-compatible LLM  
文档类型：项目开发计划 / Sprint Task Breakdown  
适用阶段：MVP 开发阶段  

---

## 1. 文档目标

本文档用于指导 MangaForge MVP 阶段的实际开发工作，明确：

- 项目阶段划分
- Sprint 排期
- 前端任务拆解
- 后端任务拆解
- AI Agent 任务拆解
- 测试与验收标准
- 里程碑交付物
- 项目风险与应对方案

MVP 的核心目标是：

> 跑通“小说章节 → AI 理解 → 单话改编 → 漫画分镜 → Prompt 生成 → 人工编辑 → 导出制作资料”的最小闭环。

---

## 2. MVP 总目标

### 2.1 核心业务闭环

MVP 必须完整支持以下流程：

```text
用户登录
↓
创建 MangaForge 项目
↓
上传 TXT 小说
↓
自动拆分章节
↓
查看 / 编辑章节
↓
AI 解析章节
↓
生成角色 / 事件 / 场景基础知识库
↓
选择 1～3 章生成一话漫画
↓
AI 生成单话大纲和漫画剧本
↓
AI 生成 30～60 格分镜
↓
用户编辑分镜
↓
复制 Prompt
↓
导出 Excel / Word / HTML / JSON / Prompt TXT
```

---

### 2.2 MVP 不做内容

MVP 阶段严格控制范围，以下内容不进入第一版开发：

```text
图片生成
LoRA 训练
多人实时协作
复杂权限系统
支付系统
版权交易
漫画平台发布
移动端 App
完整管理后台
自动排版成正式漫画图片
```

## 3. 项目阶段规划

MangaForge MVP 建议拆分为 7 个阶段。

| 阶段 | 名称 | 目标 | 周期 |
|---|---|---|---|
| 阶段 0 | 项目初始化 | 仓库、环境、工程骨架、规范 | 3～5 天 |
| 阶段 1 | 基础平台 | 登录、项目管理、基础布局 | 2 周 |
| 阶段 2 | 小说导入 | TXT 上传、自动拆章、章节管理 | 2 周 |
| 阶段 3 | AI 解析 | AI 任务系统、章节解析、知识库 | 2 周 |
| 阶段 4 | 单话改编 | 选择章节、生成漫画剧本 | 1～2 周 |
| 阶段 5 | 分镜编辑 | 分镜生成、编辑、预览、Prompt | 2 周 |
| 阶段 6 | 导出测试 | 多格式导出、测试、部署、演示 | 1～2 周 |

预计总周期：

```text
8～11 周
```

如果团队人手充足，可压缩到：

```text
6～8 周
```

## 19. 风险与应对

### 19.1 AI 输出不稳定

风险表现：

```text
JSON 格式错误
字段缺失
分镜数量不对
内容跑偏
```

应对方案：

```text
使用 JSON Schema
后端做 Pydantic 校验
保留原始输出
失败自动重试
增加 JSON 修复逻辑
提供人工编辑能力
```

---

### 19.2 小说章节拆分不准确

风险表现：

```text
章节标题识别失败
章节内容错位
番外误识别
广告段落混入
```

应对方案：

```text
支持多种章节正则
提供手动编辑
支持重新拆章
支持合并 / 拆分章节
标记异常章节
```

---

### 19.3 HermesAgent 接入不稳定

风险表现：

```text
调用失败
超时
输出格式不统一
环境部署复杂
```

应对方案：

```text
封装 Agent Adapter
设置超时和重试
保留 LLM Provider 直连备用方案
AI 任务异步执行
任务失败可重试
```

---

### 19.4 分镜编辑器开发复杂

风险表现：

```text
字段多
交互复杂
拖拽排序易出错
保存状态混乱
```

应对方案：

```text
先做卡片列表版本
拖拽排序作为 P0 但简化实现
预览作为 P1
单格重生成作为 P1
先保证编辑保存稳定
```

---

### 19.5 范围膨胀

风险表现：

```text
提前做图片生成
提前做多人协作
提前做支付
提前做复杂权限
```

应对方案：

```text
严格控制 MVP 范围
所有新增需求进入 Backlog
P0 不完成不启动 P2
每周复盘范围变化
```

```text
8. 保存角色到角色库
9. 选择第 1～3 章生成第 1 话漫画
10. 查看并编辑漫画剧本
11. 生成 40 格分镜
12. 编辑其中 3 个分镜
13. 拖拽调整分镜顺序
14. 复制全部 Prompt
15. 导出 Excel
16. 导出 Word
17. 导出 HTML
18. 下载导出文件
```

## 25. 结论

MangaForge MVP 的开发重点不是功能堆叠，而是稳定跑通核心闭环：

```text
小说上传
→ 自动拆章
→ AI 章节解析
→ 单话改编
→ 分镜生成
→ 编辑
→ 导出
```

项目推进时需要坚持：

```text
先 P0，后 P1，再 P2
先主流程，后高级功能
先结构化输出，后视觉生成
先单用户，后团队协作
先 Prompt 导出，后图片出图
```

只要 MVP 阶段能够稳定完成“从小说章节到可编辑分镜资料”的闭环，MangaForge 就具备进一步迭代为完整 AI 漫改生产平台的基础。
