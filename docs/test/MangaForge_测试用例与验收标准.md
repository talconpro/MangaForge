版本：V0.5 MVP

文档类型：Test Cases & Acceptance Criteria  
适用阶段：MVP 测试与验收阶段  

---

## 1. 文档目标

本文档用于定义 MangaForge MVP 的测试范围、测试策略、测试用例、验收标准和缺陷等级，确保系统可以稳定跑通核心业务闭环。

核心验收目标：

```text
小说上传
→ 自动拆章
→ AI 解析
→ 单话改编
→ 分镜生成
→ 分镜编辑
→ 导出制作资料
```

## 5. 缺陷等级

| 等级 | 定义 | 示例 |
|---|---|---|
| P0 | 主流程阻塞，必须立即修复 | 无法登录、无法上传、无法生成分镜 |
| P1 | 核心功能异常，当前 Sprint 修复 | AI 任务状态错误、导出字段缺失 |
| P2 | 非核心功能异常或体验问题 | 样式错位、提示不清晰 |
| P3 | 低优先级优化 | 文案优化、间距调整 |

## 6. 主流程验收用例

### TC-MAIN-001 完整主流程

| 项目 | 内容 |
|---|---|
| 用例目标 | 验证 MangaForge MVP 主流程完整可用 |
| 前置条件 | 用户已注册，系统服务正常 |
| 操作步骤 | 见下方 |
| 预期结果 | 可从小说上传完整导出分镜资料 |

#### 操作步骤

```text
1. 用户登录 MangaForge
2. 创建项目《云州旧梦》
3. 上传测试 TXT 小说
4. 系统自动拆章
5. 查看章节列表
6. 打开第 1 章详情
7. 对第 1～3 章执行 AI 解析
8. 查看章节解析结果
9. 保存角色到知识库
10. 选择第 1～3 章生成第 1 话
11. 查看本话大纲和漫画剧本
12. 保存单话
13. 生成 40 格分镜
14. 编辑其中 3 个分镜
15. 拖拽调整分镜顺序
16. 复制全部 Prompt
17. 导出 Excel
18. 导出 Word
19. 导出 HTML
20. 下载导出文件
```

#### 通过标准

```text
主流程无阻塞
核心数据无丢失
AI 失败可重试
分镜可编辑
导出文件内容完整
```

## 19. MVP 总体验收标准

### 19.1 功能验收

```text
用户可以登录。
用户可以创建项目。
用户可以上传 TXT。
系统可以自动拆章。
用户可以查看和编辑章节。
用户可以 AI 解析章节。
用户可以保存角色到知识库。
用户可以选择 1～3 章生成一话。
系统可以生成漫画剧本。
系统可以生成 30～60 格分镜。
用户可以编辑分镜。
用户可以复制 Prompt。
用户可以导出 Excel / Word / HTML / JSON / Prompt TXT。
```

### 19.2 AI 验收

```text
章节解析成功率 ≥ 90%
单话生成成功率 ≥ 90%
分镜生成成功率 ≥ 90%
JSON 格式正确率 ≥ 95%
分镜数量误差 ≤ ±10%
高级别禁改设定不得违反
```

### 19.3 导出验收

```text
导出成功率 100%
导出文件可下载
导出字段完整
HTML 可打开预览
```
